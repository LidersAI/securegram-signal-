# LIDERS CHAT — Клиент

Защищённый P2P мессенджер. Один HTML-файл без зависимостей.

**Сайт:** https://liderschat.ru  
**Сигнальный сервер:** https://liders-chat-signal.onrender.com

## Архитектура

```
Браузер A                    Сервер (Render)              Браузер B
   |                               |                           |
   |── POST /register ────────────>|                           |
   |<─ {token, peerId} ───────────|                           |
   |                               |                           |
   |── GET /poll/myId (20s hold) ─>|<── POST /poll/theirId ───|
   |<─ {signals: [offer]} ────────|                           |
   |                               |                           |
   |════════ WebRTC DataChannel (P2P, E2E шифрование) ════════|
   |                               |                           |
   |── зашифрованные сообщения ──────────────────────────────>|
   |<─ зашифрованные сообщения ───────────────────────────────|
```

Сервер используется **только** для:
1. Регистрации/входа
2. Обмена WebRTC offer/answer/ICE (сигнализация)
3. Временного хранения зашифрованных офлайн-сообщений

Сообщения передаются **напрямую** между браузерами по WebRTC.  
Сервер не может их расшифровать.

## Технологии

| Компонент | Технология |
|---|---|
| Шифрование | ECDH P-384 + AES-256-GCM |
| Сигнализация | HTTP long-polling (обход DPI) |
| P2P транспорт | WebRTC DataChannel |
| Хранилище | localStorage (`sg_v1`) |
| Аватары | localStorage (`sg_avatars`) |
| PWA | Service Worker + manifest.json |

## Файлы в репо

```
index.html          — всё приложение (HTML + CSS + JS в одном файле)
manifest.json       — PWA манифест
sw.js               — Service Worker (кэш иконок, HTML всегда с сети)
404.html            — редирект для старых ссылок /securegram/...
favicon.ico
icon-*.png          — иконки разных размеров (72–512px)
icon-*-maskable.png — maskable иконки для Android
```

## Функциональность

### Аккаунты
- Регистрация по никнейму + пароль
- Вход с любого устройства под одним никнеймом
- Анонимный режим (без регистрации, случайный ID)
- Восстановление пароля по резервному коду
- Резервный код показывается один раз при регистрации

### Чаты
- Личные сообщения (DM) по ID/никнейму
- Группы с инвайт-ссылками
- Закреплённые чаты (долгое нажатие / ПКМ)
- Поиск по чатам и сообщениям
- Удаление чатов

### Сообщения
- Текст, фото, файлы (до 10MB), голосовые
- Реакции, ответы, редактирование, удаление
- Пересылка, закреплённые сообщения в группе
- Статус прочтения (✓✓)
- Индикатор набора текста

### Группы
- Создание с иконкой и пин-кодом
- Приглашение по ссылке
- Добавление участников по ID
- Удаление участников
- Переименование группы
- Список участников с онлайн-статусом

### Профиль
- Карточка участника (имя, телефон, общие группы)
- Аватар (фото, сжимается до 80×80)
- Переименование контакта
- Перенос аккаунта на другое устройство (экспорт/импорт ключа)

### Звонки
- Аудио и видеозвонки (WebRTC)
- Только в личных чатах

### Безопасность
- E2E шифрование всех сообщений
- Приватный ключ только в памяти браузера, нигде не сохраняется
- Новый ключ при каждом открытии
- Защита от replay-атак (seen message IDs)

### UX
- Тёмная и светлая тема
- PWA — устанавливается на главный экран
- Свайп влево/вправо для навигации на мобильном
- Офлайн-доставка через relay сервер (сообщения доходят когда получатель офлайн)
- Автопереподключение при потере сети

## Хранилище (localStorage)

```js
sg_v1: {
  myId,           // peer ID (никнейм или случайный)
  myName,         // отображаемое имя
  myPhone,        // телефон (необязательно)
  rooms: {        // группы
    [id]: { id, name, emoji, pin, members, messages, isHost, pinned }
  },
  dms: {          // личные чаты
    [peerId]: { name, messages, explicit }
  },
  pinned: {       // закреплённые чаты
    [chatKey]: true
  }
}
sg_theme:   'dark' | 'light'
sg_avatars: { [peerId]: base64 }
lc_token:   токен сессии (если есть аккаунт)
lc_username: никнейм (если есть аккаунт)
```

## Известные ограничения

- Аккаунты на Render Free могут сброситься при пересоздании контейнера
- История сообщений только в localStorage — при очистке браузера пропадёт
- Звонки требуют TURN серверов в некоторых сетях
- WebSocket может блокироваться провайдером — используем HTTP polling как обход
